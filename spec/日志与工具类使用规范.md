# 日志与工具类使用规范

## 1. 日志规范

### 1.1 日志框架

- 使用 Log4j2（已排除 Spring Boot 默认的 Logging）
- 通过自定义 `LogUtil` 工具类封装 SLF4J

### 1.2 日志调用方式

```java
// info 级别
LogUtil.info(FilmService.class, "--> 新增影片：" + film.getName());

// error 级别
LogUtil.error(FilmService.class, "更新失败，id=" + id);

// 不带类名参数的 error
LogUtil.error("系统异常：" + e.getMessage());

// 带异常对象的 error（推荐，可打印堆栈）
LogUtil.error(FilmService.class, e.getMessage(), e);
```

### 1.3 日志格式

- 消息前缀统一加 `--> `
- 日志内容清晰描述操作和关键数据

### 1.4 禁止事项

- **禁止**使用 `System.out.println()` 输出日志
- **禁止**使用 `e.printStackTrace()` 打印异常

## 2. 核心工具类

### 2.1 ResponseReturnUtil — 响应构建

```java

// 失败响应

ResponseReturnUtil.returnErrorMsgAndData("消息", data)    // 失败 + 消息 + 数据

// 写入 HttpServletResponse（供拦截器使用）

// 预定义常量
```

## 3. 分页计算模式

Service 层手动计算分页偏移量：

```java
int pc = 1;   // 当前页码
int ps = 10;  // 每页大小

int offset = (pc - 1) * ps;

Map<String, Object> param = new HashMap<>(4);
param.put("start", offset);
param.put("count", ps);
```

## 4. 项目实际情况（与规范不一致之处）

以下为项目中实际存在但**与规范冲突**的情况，编码时注意区分：

| 规范要求 | 实际违反的代码 |
|---------|--------------|
| 统一使用 Fastjson | `AuthenticationController` 使用了 `net.sf.json.JSONObject` 和 `org.json.JSONException` |
| 统一使用 Fastjson | `JSONUtil` 完全基于 `net.sf.json` |
| 禁止 `System.out.println` | `Util`、`AuthenticationController`、`Tools`、`MD5Auth` 中均有使用 |
| 禁止 `e.printStackTrace()` | `DateUtil`、`Tools`、`FileOperate`、`CodeConvert`、`ImageHelper` 等多个类存在 |
| 禁止 `StringBuffer` | `Tools`、`MD5`、`GeetestLib` 中使用 `StringBuffer` |
| 使用 `@Resource` | `MySolrConfig` 使用了 `@Autowired` |
| Interceptor 继承 `HandlerInterceptorAdapter` | 实际实现 `HandlerInterceptor` 接口 |

- 集合初始化时指定预期容量：`new HashMap<>(4)`、`new ArrayList<>()`
- 字符串拼接使用 `StringBuilder`（**不**使用 `StringBuffer`）
- 统一使用 Fastjson（`com.alibaba.fastjson`），**不**混用其他 JSON 库
