# 数据库规范

## 1. 命名规范

### 1.1 表名
- 前缀：`t_`
- 蛇形命名（snake_case），单数名词
- 示例：`t_film`、`t_user`、`t_comment`、`t_view_history`、`t_bullet`

### 1.2 字段名
- 蛇形命名（snake_case）
- 示例：`type_id`、`view_number`、`add_time`、`comment_number`、`sub_class_id`

### 1.3 主键
- 32 位大写 UUID 字符串（非自增）
- 字段名统一为 `id`
- 通过 `Tools.UUID()` 生成

## 2. 引擎与编码

```sql
ENGINE=InnoDB DEFAULT CHARSET=utf8
```

## 3. 逻辑删除

- 使用 `isUse` 字段表示逻辑删除/启用状态
- 值为 `0`（禁用/删除）或 `1`（启用）
- 所有查询默认过滤 `is_use = 1`

## 4. 常用字段模式

```sql
`id`          VARCHAR(32)  NOT NULL COMMENT 'UUID主键',
`name`        VARCHAR(255)          COMMENT '名称',
`is_use`      INT(1)       DEFAULT 1 COMMENT '是否启用：0-禁用，1-启用',
`add_time`    DATETIME              COMMENT '添加时间',
`order_no`    INT(11)               COMMENT '排序号',
`view_number` INT(11)      DEFAULT 0 COMMENT '观看次数',
```

## 5. MyBatis XML 对应规范

- 表名和字段名使用反引号 `` ` `` 包裹
- 实体字段使用小驼峰映射到数据库蛇形字段

```xml
<sql id="filmColumn">
    `id`, `name`, `image`, `url`, `view_number`, `comment_number`,
    `is_use`, `add_time`, `type_id`, `cata_log_id`,
    `sub_class_id`, `decade_id`, `loc_id`, `level_id`, `raty_id`
</sql>
```
