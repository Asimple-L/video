# 单元测试编写规范

## 1. 测试框架

- 使用 JUnit 4 + SpringBootTest
- 运行器：`SpringJUnit4ClassRunner`

## 2. 测试类定义

### 2.1 基本样板

```java
@RunWith(SpringJUnit4ClassRunner.class)
@SpringBootTest(classes = App.class, webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class ManagerControllerTest {

    @Resource
    private ManagerController controller;

    @Test
    public void testMethod() {
        // ...
    }
}
```

### 2.2 类放置位置

- Controller 测试：`src/test/java/com/asimple/controller/XxxControllerTest.java`
- 实体/DTO 测试：`src/test/java/com/asimple/dto/DtoTest.java`

## 3. 测试方法

### 3.1 方法命名

- 以 `test` 开头，后接被测试方法名（PascalCase）
- 如测试 `login` 方法，方法名为 `testLogin`

### 3.2 断言

使用 JUnit `Assert` 类，**不使用** `System.out.println` 做断言：

```java
Assert.assertNotNull(result);
Assert.assertEquals("000000", result.getString("code"));
Assert.assertTrue(result.getBoolean("success"));
```

## 4. 完整示例

### 4.1 Controller 测试

```java
@RunWith(SpringJUnit4ClassRunner.class)
@SpringBootTest(classes = App.class, webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class ManagerControllerTest {

    @Resource
    private ManagerController managerController;

    @Test
    public void testFilmType() {
        JSONObject result = (JSONObject) managerController.filmType();
        Assert.assertNotNull(result);
        Assert.assertEquals("000000", result.getString("code"));
    }

    @Test
    public void testFilmList() {
        JSONObject result = (JSONObject) managerController.filmList(1, 10);
        Assert.assertNotNull(result);
        Assert.assertEquals("000000", result.getString("code"));
    }
}
```

### 4.2 实体测试

```java
@RunWith(SpringJUnit4ClassRunner.class)
@SpringBootTest(classes = App.class, webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class DtoTest {

    @Test
    public void testDto() {
        Film film = new Film();
        film.setName("测试影片");
        Assert.assertEquals("测试影片", film.getName());

        User user = new User();
        user.setUsername("admin");
        Assert.assertEquals("admin", user.getUsername());
    }
}
```

## 5. 运行测试

```bash
# 运行全部测试
mvn test

# 运行单个测试类
mvn test -Dtest=ManagerControllerTest

# 运行单个测试方法
mvn test -Dtest=ManagerControllerTest#testMethod

# 指定 profile 测试
mvn test -Dspring.profiles.active=dev
```

## 6. 规范要求

- 每个 Controller 类应有对应的测试类
- 测试方法覆盖正常流程和异常流程
- 使用 `@Resource` 注入被测试类（**不**使用 `@Autowired`）
- 测试类构造函数和字段可自由定义，不使用 Lombok 也可
- 不要依赖测试执行顺序
