# 缓存与定时任务规范

## 一、Redis 缓存

### 1. 技术选型

- 使用 `spring-boot-starter-redis`
- 通过 Spring 缓存注解使用（`@Cacheable` / `@CacheEvict`）
- 配置类：`RedisCacheConfig`

### 2. 读操作 — @Cacheable

在 Service 方法上加 `@Cacheable`，自动缓存返回结果：

```java
@Cacheable(value = "film", key = "'film_' + #pc + '_' + #ps")
public PageBean<Film> getFilmList(int pc, int ps) {
    return ...;
}
```

- `value` — 缓存分区名称，**项目中直接使用字符串字面量**（未统一使用常量）
- `key` — 使用 SpEL 表达式，确保唯一性
- 方法第一次执行时缓存结果，后续直接返回缓存值

### 3. 写操作 — @CacheEvict

```java
@CacheEvict(value = "film", allEntries = true)
public boolean addFilm(Film film) {
    return filmMapper.add(film);
}

@CacheEvict(value = "film", allEntries = true)
public boolean updateFilm(Film film) {
    return filmMapper.edit(film);
}

@CacheEvict(value = "film", allEntries = true)
public boolean deleteFilm(String id) {
    return filmMapper.delete(id);
}
```

- `allEntries = true` — 清空整个缓存分区（避免逐条清除的遗漏）
- 新增、修改、删除操作都要加 `@CacheEvict`

### 4. 缓存分区名称对照

项目中实际使用的缓存 value 名称：

| 缓存 value | 用途 |
|-----------|------|
| `"film"` | 影片分页列表 |
| `"user"` | 用户信息 |
| `"redis_cataLogList"` | 分类列表 |
| `"redis_levelList"` | 等级列表 |
| `"redis_decadeList"` | 年代列表 |
| `"redis_locList"` | 地区列表 |
| `"index_filmTuijian"` | 首页推荐 |
| `"index_filmPaiHang"` | 首页排行榜 |

### 5. 多分区清理

`@CacheEvict` 支持同时清理多个缓存分区：

```java
@CacheEvict(value = {"index_filmTuijian", "index_filmPaiHang"}, allEntries = true)
public boolean update(FilmUpdateInfo film) {
    return filmMapper.updateFilm(film) == 1;
}
```

### 6. 缓存 Key 常量

定义在 `VideoKeyNameUtil` 中（还包含 Session Key、编码等）：

```java
public class VideoKeyNameUtil {
    // 缓存分区
    public static final String FILM = "film";
    public static final String USER = "user";

    // Session Key
    public static final String ADMIN_USER_KEY = "adminUser";
    public static final String USER_KEY = "u_sk1";

    // 加密
    public static final String PASSWORD_KEY = "a1s2i3m4p5l6e7";

    // 编码
    public static final String ENCODE = "UTF-8";

    // 分隔符
    public static final String RES_NAME_SPLIT = "@@";
    public static final String EPISODES_NAME_SPLIT = "##";
}
```

### 7. 缓存配置（RedisTemplate 序列化）

```java
@Configuration
@EnableCaching
public class RedisCacheConfig extends CachingConfigurerSupport {

    @Bean
    public KeyGenerator keyGenerator() {
        return (target, method, params) -> {
            StringBuilder sb = new StringBuilder();
            sb.append(target.getClass().getName());
            sb.append(method.getName());
            for (Object obj : params) {
                sb.append(obj.toString());
            }
            return sb.toString();
        };
    }

    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
        template.setHashKeySerializer(new StringRedisSerializer());
        template.setHashValueSerializer(new GenericJackson2JsonRedisSerializer());
        template.afterPropertiesSet();
        return template;
    }
}
```

### 8. 注意

- 项目中的缓存分区名称**并未统一使用常量**，直接使用字符串字面量（如 `"redis_cataLogList"`、`"index_filmTuijian"`）
- 新增缓存时请先查表确认分区名称，保持与现有命名风格一致

## 二、定时任务

### 1. 类定义

- 类加 `@Component` + `@Configuration` + `@EnableScheduling`
- 方法加 `@Scheduled(cron = "...")`

```java
@Component
@Configuration
@EnableScheduling
public class RankTask {

    @Resource
    private FilmService filmService;

    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点执行
    public void rank() {
        // 定时任务逻辑
    }
}
```

### 2. Cron 表达式示例

| 表达式 | 含义 |
|--------|------|
| `0 0 2 * * ?` | 每天凌晨 2 点 |
| `0 0/30 * * * ?` | 每 30 分钟 |
| `0 0 0/1 * * ?` | 每小时 |
| `0 0 0 * * ?` | 每天午夜 |

### 3. 现有定时任务

#### RankTask（排行榜任务）
- 每天凌晨 2 点执行
- 更新影片排行榜数据

#### SolrTask（Solr 索引任务）
- 定时同步数据库到 Solr 索引
- 确保搜索数据的一致性
