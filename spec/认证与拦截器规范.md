# 认证与拦截器规范

## 1. 认证方式

- 基于 Session 认证
- **没有**使用 Spring Security
- 使用 `RequestUtil` 工具类手动校验

## 2. RequestUtil 鉴权方法

```java
// 判断是否登录
RequestUtil.isLogin(request)              // 返回 boolean

// 判断是否未以管理员身份登录
RequestUtil.isNotAdminLogin(request)      // 返回 boolean

// 判断是否非本人操作
RequestUtil.isNotSelfLogin(request, id)   // 返回 boolean
```

## 3. 拦截器

### 3.1 ProfileInterceptor（用户鉴权）

- 拦截用户相关请求（Profile 路径）
- 检查用户是否已登录
- 实现 `HandlerInterceptor` 接口（**非**继承 `HandlerInterceptorAdapter`）

```java
public class ProfileInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler) throws Exception {
        if (RequestUtil.isNotLogin(request)) {
            JSONObject jsonObject = ResponseReturnUtil.returnErrorWithMsg("请先登录");
            ResponseReturnUtil.returnJson(response, jsonObject.toJSONString());
            return false;
        }
        return true;
    }
}
```

- 拦截器中通过 `ResponseReturnUtil.returnJson()` 直接写入 JSON 响应
- 部分请求路径放行白名单：`/getType`、`/getSubClass` 等

### 3.2 AdminUserInterceptor（管理鉴权）

- 拦截管理相关请求（`/admin/**` 路径）
- 检查当前用户是否为管理员
- 实现 `HandlerInterceptor` 接口

```java
public class AdminUserInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler) throws Exception {
        if (RequestUtil.isNotAdminLogin(request)) {
            JSONObject jsonObject = ResponseReturnUtil.returnErrorWithMsg("请登录管理员账号!");
            ResponseReturnUtil.returnJson(response, jsonObject.toJSONString());
            return false;
        }
        return true;
    }
}
```

## 4. 拦截器注册

在 `MyWebAppConfigurer` 中注册拦截器（注意：`AdminUserInterceptor` 拦截的是 `/admin/**` 而非 `/manager/**`）：

```java
@Configuration
public class MyWebAppConfigurer extends WebMvcConfigurerAdapter {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        // 用户鉴权拦截器 - 拦截 /profile/** 但排除部分公开路径
        registry.addInterceptor(new ProfileInterceptor())
                .addPathPatterns("/profile/**");

        // 管理员鉴权拦截器 - 拦截 /admin/**
        registry.addInterceptor(new AdminUserInterceptor())
                .addPathPatterns("/admin/**");
    }
}
```

> 注意：`AdminUserInterceptor` 注册路径为 `/admin/**`，与 `ManagerController` 的类级 `@RequestMapping("/admin")` 对应。

## 5. Controller 中的鉴权模式

在每个需要登录的方法开头手动校验：

```java
@RequestMapping(value = "/userInfo", method = RequestMethod.POST, produces = "application/json;charset=UTF-8")
public Object userInfo(HttpServletRequest request) {
    if (!RequestUtil.isLogin(request)) {
        return ResponseReturnUtil.returnErrorWithMsg(ResponseReturnUtil.LOGIN_FIRST);
    }
    // 业务逻辑...
}
```

## 6. 获取当前用户

通过继承 `CommonController` 获取用户信息：

```java
public class CommonController {
    // 获取当前登录用户
    protected User getUserInfo(HttpServletRequest request) { ... }

    // 获取当前管理员用户
    protected User getAdminUserInfo(HttpServletRequest request) { ... }
}
```

## 7. ResponseReturnUtil.returnJson()

拦截器可直接向 `HttpServletResponse` 写入 JSON 错误信息：

```java
ResponseReturnUtil.returnJson(response, jsonObject.toJSONString());
```

该方法设置 UTF-8 编码和 `text/html` 内容类型，通过 `PrintWriter` 输出。

## 8. 拦截器白名单

拦截器可在 `preHandle` 中通过请求 URL 判断是否放行某些路径：

```java
// ProfileInterceptor 中绕过登录检查的路径示例
String uri = request.getRequestURI();
if (uri.contains("/getType") || uri.contains("/getSubClass")) {
    return true;  // 公开接口，跳过鉴权
}
```

## 9. 鉴权常量

定义在 `ResponseReturnUtil` 中：

```java
public static final String LOGIN_FIRST = "请先登录";
public static final String ADMIN_LOGIN_FIRST = "请以管理员身份登录";
public static final String ILLEGAL_VISIT = "非法访问";
```
