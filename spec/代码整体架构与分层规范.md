## 3. 请求处理流程

```
客户端请求
    → Interceptor（ProfileInterceptor / AdminUserInterceptor）
    → Controller（参数绑定 + 鉴权 + 调用 Service）
    → Service（业务逻辑 + 缓存 + 调用 Mapper）
    → Mapper Interface（DAO 定义）
    → MyBatis XML（SQL 执行）
    → MySQL Database
    → 返回 JSONObject 响应
```

## 4. 分层职责

### Controller 层
- 接收 HTTP 请求、参数绑定
- 鉴权校验（`RequestUtil`）
- 调用 Service 层
- 构建统一 JSON 响应（`ResponseReturnUtil`）

### Service 层
- 业务逻辑编排
- 缓存管理（`@Cacheable` / `@CacheEvict`）
- 事务管理（`@Transactional`）
- 调用 Mapper 进行数据库操作

### Mapper 层
- 继承 `IBaseDao<T>` 获得通用 CRUD
- 定义自定义查询方法
- 通过 XML 映射文件实现 SQL

## 7. Maven 注意

- 打包命令：
  - 开发：`mvn clean package -DskipTests`
  - 生产：`mvn clean package -P prod`
- POM 中部分依赖使用版本范围（如 `[1.2.31,)`），可能引入不兼容版本
- 无专门的 lint/checkstyle 插件
