# 实体类编写规范

## 1. 类定义

- 使用 Lombok `@Data` 注解
- 实现 `Serializable` 接口
- 类名：单数名词，PascalCase（如 `Film`、`User`、`Bullet`）

```java
@Data
public class Film implements Serializable {
}
```

## 2. 字段命名

- Java 字段理论上使用小驼峰（camelCase），但项目中大量实体直接使用 snake_case 与数据库列名一致
- 数据库列名使用蛇形命名（snake_case）
- 混合风格示例（同一实体中可混用两种风格）：

```java
// camelCase 风格
private String cataLogId;
private Integer viewNumber;
private Date addTime;

// snake_case 风格（项目中同样常见）
private String type_id;
private String subClass_id;
private Integer view_number;
```

> 注意：编码前请查看同类实体，保持与相邻实体风格一致。`Film` 实体同时使用了两种风格。

## 3. 主键

- 32 位大写 UUID 字符串（非自增），通过 `Tools.UUID()` 生成

```java
private String id;
```

## 4. 常用字段

- `isUse` — 逻辑删除/启用状态（值为 0 或 1）

```java
private Integer isUse;
```

## 5. 关联对象

### 5.1 单对象关联

实体中可以包含关联的实体对象用于级联展示：

```java
// 影片分类关联
private CataLog cataLog;

// 影片子分类关联
private SubClass subClass;

// 影片类型关联
private Type type;
```

### 5.2 集合关联

实体中可以包含 `List` 集合用于一对多关联展示：

```java
// 分类下包含子分类列表
private List<SubClass> subClassList;

// 子分类下包含类型列表
private List<Type> types;

// 影片包含资源列表
private List<Res> res;
```

## 6. Solr 字段映射

当实体参与 Solr 搜索时，在字段上加 `@Field` 注解（`org.apache.solr.client.solrj.beans.Field`）：

```java
@Field
private String id;

@Field("video_film_name")
private String name;

@Field("video_film_image")
private String image;
```

Solr 字段命名约定：`video_` + 业务描述（如 `video_film_name`、`video_film_onDecade`）。

## 7. DTO 实体

对于非数据表的纯传输对象，不要使用 `@Data`，使用手动 getter/setter。如 `FilmUpdateInfo`：

```java
public class FilmUpdateInfo implements Serializable {
    private String id;
    private String type_id;
    private String loc_id;
    private String cataLog_id;
    private String subClass_id;
    // 手动 getter/setter...
}
```

## 8. 完整示例

```java
@Data
public class Film implements Serializable {

    private String id;                    // UUID 主键
    private String name;                  // 影片名称
    private String image;                 // 封面图
    private String url;                   // 播放地址
    private String typeId;                // 类型ID
    private String subClassId;            // 子分类ID
    private String cataLogId;             // 分类ID
    private Integer viewNumber;           // 观看次数
    private Integer commentNumber;        // 评论数
    private Integer isUse;                // 逻辑删除标志
    private Date addTime;                 // 添加时间
    private String decadeId;              // 年代ID
    private String locId;                 // 地区ID
    private String levelId;               // 等级ID
    private String ratyId;                // 评分ID

    // 关联对象（用于级联查询展示）
    private CataLog cataLog;
    private SubClass subClass;
    private Type type;
    private Decade decade;
    private Loc loc;
    private Level level;
    private Raty raty;
}
```

## 9. 特别说明

- 实体可以同时有 `@Data` 和手动构造器（项目中有此先例）
- 保持实体纯净，**不要**在实体中添加业务逻辑
- `Date` 类型字段使用 `java.util.Date`
- 如果字段需要做格式转换，可使用 `@JsonFormat` 注解
- 不需要为每个字段编写 Javadoc，但类级别建议有中文注释说明用途
